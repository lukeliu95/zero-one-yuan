Process

从一句价格表，长成一部小说。

项目状态显示 A/B/C/D 四阶段已完成。生成过程不是“直接写正文”，而是先把主题、验收标准、世界规则和人物动作固定，再通过 9 轮 Phase D 把正文补全、抛光、审计，并完成 DeepSeek V5 的真实模型锚定。

Phase A

需求工程

把“0.1 元 / 1M token”从科技新闻变成小说母题。

  • 5 轮 discovery interview
  • 5 个目标与验收标准
  • 三幕阅读节奏与情感落点
Phase B

创作骨架

建立世界观、人物档、分场大纲，让正文有可追溯的骨架。

  • 水位线经济学
  • 三主角弧线与招牌动作
  • 13 章分场大纲
Phase C

正文与质检

先写首批章节，再按主题、角色、概念融合、文学性和结构完成质量评估。

  • 第 1-4 章基础正文
  • 质量评分 85.15
  • 识别妻子戏份与主题词缺口
Phase D

产品进化

9 轮 CEO round 逐步补齐：妻子戏、术语迁移、全书扩写、章节抛光、事实纠错、一致性审计与 DeepSeek V5 模型重锚。

  • 第 5-13 章一次性闭合
  • Claude Code / Manus 职能纠偏
  • DeepSeek V5 真实模型锚定

Worldbuilding

世界观的核心，是一条价格形成的水位线。

故事发生在 2026 年 10 月 29 日到 2027 年 4 月 28 日。DeepSeek V5 把推理价格打到 0.1 元 / 百万 token，同价、长上下文、能力超越公开基线，随后软件行业的估值、岗位和尊严开始重新定价。

0.1
RMB / 1M tokens · the waterline
沉下去：IDE、横切 SaaS、代码工厂、高级工程师岗位 浮上来：行业 agent、skill marketplace、agent observability、遗产 curator

新职业

skill writer、context engineer、taste curator、agent-ops、讣告 writer。入行变低，天花板反而更依赖审美和耐心。

新基础设施

个人 token 钱包、reasoning gas 期货、agent observability SaaS，以及把 skill 当商品出售的市场。

新法律

代码遗产法案与 Bounded Autonomy 审计标准，让“代码会不会被记住”从私人问题变成公共制度。

Characters

三个人，三种活法。

人物不是概念的传声筒。每个人都有自己的动作、物件和迟到的答案。

林行

高级工程师 → 临终关怀讣告 writer

他没有恨 agent。他只是慢慢承认，自己用三年的 Claude Code 从伙伴变成 co-reviewer，再变成替代者。

答案：愿意慢的能力。最终把“最后一次 code review”写成讣告。

姜禾

19 岁 skill writer · 古琴与约束

她不写代码，但会写边界。她把 agent 当同事，也知道有些电话只能自己打。

答案：审美。skill 的品味决定 agent 输出的天花板。

陆衡

SaaS 创始人 → 殡葬行业 agent

他从估值崩塌里回到县城殡仪馆，重新发现行业潜规则是通用模型抓不到的数据。

答案：数据。软件没死，死的是把软件当工厂的那种活法。
