## 1. "水位线"经济学

### 1.1 0.1 元的成本拆解

把 1 百万 token 0.1 元摊开,大致是这样一杯水:

| 项 | 占比 | 量级 |
|---|---|---|
| 推理用电 | 38% | 0.038 元 |
| GPU 折旧摊销 | 22% | 0.022 元 |
| 数据中心冷却 | 14% | 0.014 元 |
| 网络出口 | 9% | 0.009 元 |
| 模型训练摊销(50 年线性) | 7% | 0.007 元 |
| 安全 / 合规 / 审计 | 6% | 0.006 元 |
| 实验室毛利 | 4% | 0.004 元 |

一杯农夫山泉零售价 2 元,出厂价约 0.4 元,水本身约 0.07 元。
**0.1 元 / 1M token ≈ 1.4 杯农夫山泉的水。** 这是书里"水位线"一词的字面来源。

### 1.2 沉降的资产类别

水位线之下,以下资产类别在 Day 0 - Day 90 内估值至少打到 1/5:

- 通用 IDE 厂商(Cursor / Windsurf 等)
- 横切面 SaaS(客户支持 / CRM / 工单系统)
- 通用代码生成工具
- 高级软件工程师岗位
- 外包代码工厂
- 老 prompt engineering 培训机构

水位线之上的新形态:

- 行业专用 agent(殡葬 / 公证 / 老年医疗 / 农业灌溉 / 物流潜规则)
- skill marketplace(skill 作为可交易资产,定价 5-200 元一份)
- agent observability SaaS(企业刚需,因合规)
- "model whisperer"(为大客户调教模型 personality 的私人服务,按小时计费)
- 注意力托管服务(替人决定哪些 agent 输出值得看)

## 3. 新基础设施

### 3.1 个人 token 钱包

一种类似支付宝的应用,余额单位是 token,而非人民币。
普通人月均充值约 50 元 = 500 百万 token,够用一整月的 agent 调用。
重度用户(skill writer)月均 500-2000 元。
**充值返赠机制**:充 100 送 20,但赠送 token 限定用于 DeepSeek V5 系列 — 巩固生态。

### 3.2 skill marketplace

类似一个混合了 GitHub Gist、Substack、淘宝的平台。
- 5 元 skill:基础工具类(待办应用 / 笔记应用)
- 30 元 skill:中型(垂直行业小工具)
- 200 元 skill:含调教过的 GroupChat 配置 + memory schema
- 1000 元+:含数据护城河指引(陆衡的殡葬 skill 在这一档)

平台抽成 15%,作者得 85%(刻意比 App Store 慷慨,以吸引创作者迁移)。

### 3.3 reasoning gas 期货

类比石油期货,但标的是"未来 6 个月某模型 1B token 的均价"。
2026 年 11 月开盘,芝加哥商品交易所首次挂牌。
本书第 7 章陆衡曾经考虑做空 reasoning gas 来对冲他的 SaaS 估值下滑,
最终没做——因为他的天使投资人 SMS 给他一句:"你不是搞金融的。"

### 3.4 agent observability SaaS

合规倒逼出来的刚需。
所有商用 agent 必须留三类追踪:
- **Audit ID**:每个决策一个 UUID,可追溯到 skill 版本 + 模型版本 + token 数
- **Escalation Path**:何时升级到人,何时拒答,何时降级到更小模型
- **Cost Ledger**:每次调用的成本(精确到 0.0001 元)

代表厂商:某家美国公司 + 国内一家拆分自前 SRE 团队的初创(本书第 5 章郑总用的就是这一家)。

### 3.5 推理基础电费补贴

2026 年 12 月,某省政府试点"推理算力券",
按个人户籍每年发 200 元 token 补贴。
舆论分化:支持者说这是"新时代水电",反对者说这是"软件失业的遮羞布"。
本书第 11 章林行做临终关怀工作,
病房里给老人申请这笔补贴,因为老人想用 Lyra-Curator 整理代码遗产。

## 4. 监管与法律

### 4.1 代码遗产法案(讨论中,Day 60 起进入立法议程)

核心条款草案:
- 任何在 GitHub 等公共平台累计获得 ≥ 1000 star 的开源仓库,作者去世后,
  其代码被纳入"数字文化遗产"清单。
- 国家或社会机构可申请将其转交"代码遗产基金会"维护,而非任由其 archived。
- 维护方有义务为遗产代码生成"配得上被记住的部分"摘要 — 用 Lyra-Curator 类工具。

这是本书第 13 章的法律背景。

### 4.2 Bounded Autonomy 强制审计标准(GB/T 草案)

国家标准化管理委员会拟于 2027 Q1 出台 GB/T 4xxxx-2027《商用智能体边界自主性技术要求》。
草案要点:
- 任何商用 agent,在"涉及金钱 / 涉及客户隐私 / 涉及法律责任"三类场景中,
  必须设置升级路径(escalation path)且至少 1 名持证人类签字。
- 审计追踪保留期 ≥ 5 年。
- 拒绝接受 agent 全自主决策的"deterministic-only"模式。

本书第 6 章 skill 学校第一节课就叫《Bounded Autonomy 入门》,
讲师在白板上写的第一句话是:"自主不是自由,自主是一种契约。"

### 4.3 数据护城河之争(司法判例)

2027 年 1 月,某地法院首次受理"行业潜规则数据"侵权案。
被告:某通用 agent 厂商,被指控"诱导从业者上传行业潜规则"以训练。
原告:某行业协会(殡葬业)。
本书第 7 章陆衡作为原告方专家证人出庭——他赢得了官司,
但官司赢的不是 agent 厂商,而是他自己 SaaS 公司的市场地位。
