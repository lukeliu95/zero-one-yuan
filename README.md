# 《零点一元》

> 副标题候选:推理崩塌后的六个月

## 为什么叫《零点一元》

主题里最锋利的那根刺,不是"LLM 比 Opus 5 还强",而是**100 万 token 只要 0.1 元人民币**——
一杯水的钱,买的是一座 1990 年代谷歌数据中心的吞吐量。

"零点一元"不是一个数字,是一道**水位线**。
水位线之下,过去三十年的软件行业全部沉了底:
- IDE、SaaS、岗位、估值、护城河、技术债——通通失重。
- 沉下去的同时,水面之上长出新东西:用自然语言开公司,用 skill 写产品,
  一个人养十万用户,推理代替了执行。

书名只有四个字,但每个字都是一笔账:
**零** = 边际成本归零
**点一** = 不是真零,还有最后一道门槛,谁能跨过谁活下来
**元** = 这是钱的故事,但更是"元"问题——什么是软件、什么是工作、什么是人的位置

## 一句话梗概

2026 年 10 月 29 日,某家实验室公开了一份 API 价格表。
六个月后,世界上最后一行手写的产品代码被 commit。
一个被裁的高级工程师、一个 19 岁的"skill writer"、
一个还没倒闭的 SaaS 创始人,在水位线两侧各自找活法。

## 三幕大纲(共 12 章 · 当前先交付第 1 章)

> **2026 真实概念锚点**(全网搜来,非杜撰)
> 多 agent 协作(AutoGen v0.4/AG2 的 GroupChat 模式) · Bounded Autonomy(有边界的自主 + 升级路径 + 审计追踪) ·
> agent 长期记忆机制(2026 关键技术突破) · Browser agent / 行业专用 agent · skill-driven / context engineering 新岗位 ·
> Manus / ChatGPT Agent / Claude Code 三家代表产品 · agent observability(监控工具成刚需) · 数据护城河之争 ·
> Gartner 预测 2026 年底 40% 企业应用集成任务型 agent · 80% 工作场景应用嵌入 agent · 15% 决策由 agent 自主完成

### 第一幕 · 水位线落下(Day 0 — Day 30)

- **第 1 章 · 价格表**(已交付)
  价格表公开当夜,Slack 八分钟零回复,会议第十一分钟陷入沉默。
- **第 2 章 · 最后一次代码评审**(本轮交付)
  林行评审一个实习生 PR,屏幕另一边他用了 3 年的 Claude Code 已经作为 co-reviewer
  并行评了同一份 PR——评论数 3 倍,响应时间 1/5。林行写到一半,把鼠标从键盘上挪开。
  他每天都用的工具,昨天还是他的"伙伴",今天起是替他完成本职工作的那位。
- **第 3 章 · 19 岁的姜禾**
  姜禾在 B 站直播"用 skill 开公司"。她的工作流是:写一份 1500 字的 skill → 提交到一个叫
  *AG2-Atelier* 的多 agent 协作框架 → 12 个角色化 agent 同时跑(产品/设计/前端/后端/QA/法务/客服/运营)→
  4 小时后产品上线。她不会写代码,但她**会写约束**——她管这个叫 "context engineering"。
- **第 4 章 · 影子工程师**
  林行公司的工程团队悄悄接入了 LangGraph 做的内部 agent,负责"影子值班"。
  本来是辅助,三周后林行才发现:工单里 73% 已经是 agent 处理完的,工程师只盖章。

### 第二幕 · 沉降(Day 30 — Day 120)

- **第 5 章 · 裁员邮件**
  SaaS 公司一夜裁掉工程团队 80%。CTO 告别信:"我们没输给 AI,我们输给了 0.1 元。"
  附件是公司接入 agent observability 平台后的成本曲线——同样的 SLA,人力 vs agent 是 47:1。
- **第 6 章 · skill 学校**
  林行面试"skill 训练营"讲师。教室里坐的全是 50 岁以上的中层管理,
  他们捏着过去十年的 OKR 当简历。课程表第一节叫 *Bounded Autonomy 入门*——
  学怎么给 agent 划边界、写升级路径、留审计追踪,而不是学 prompt。
- **第 7 章 · 创始人陆衡**
  陆衡的 SaaS 估值从 2 亿掉到 200 万。他不裁员,他改商业模式:
  把产品改成"一份 9.9 元就能买走的 skill",再卖一个垂直行业(殡葬业)的专用 agent。
  他赌的是"通用 agent 永远做不进具体行业的潜规则"。
- **第 8 章 · 软件葬礼**
  三人在一场线下"软件葬礼"相遇。有人朗读 stripe-go 的 README 像念悼词。
  最后一项议程:把 GitHub 上 *archived* 状态的 repo 数实时投影到墙上——
  当晚增加 18 万个。
- **第 9 章 · 范式之争**
  推理便宜之后,真正的护城河变成什么?三人三答:
  陆衡说**数据**(尤其是行业里那种"不公开但每个老人都知道"的潜规则);
  姜禾说**审美**(skill 的品味决定 agent 输出的天花板);
  林行第一次开口:"会不会是,**愿意慢的能力**?"

### 第三幕 · 新水面(Day 120 — Day 180)

- **第 10 章 · 一个人公司**
  姜禾的公司只有她一个人,12 万付费用户。她每天工作 4 小时,其余时间练古琴。
  她公司的"组织架构图"是一份 GroupChat 配置——14 个 agent,每个有 memory namespace,
  每周日她花 1 小时给它们做 retro。"我不是在偷懒,我在还债——我们这代人欠自己的注意力。"
- **第 11 章 · 林行的转身**
  林行没做 skill writer,也没创业。他去了一家做"AI 不能做的事"的公司——临终关怀。
  面试官问他为什么离开软件行业,他说:"我没离开,我只是不再做 agent 替我做的部分。"
- **第 12 章 · 陆衡的反扑**
  陆衡的殡葬专用 agent 半年做到 ARR 5000 万。采访里他说:
  "Gartner 那个 40% 数据是平均数。垂直行业的真实数字是 92%。
   软件没死,死的是把软件当工厂的那种活法。"
- **第 13 章(尾声) · 第 181 天**
  价格表公开 180 天后的清晨,林行在病房里给一个老人写讣告。
  老人是上世纪 80 年代某个开源项目的核心维护者。他床头放着一台老 ThinkPad,
  屏幕上还开着一个 agent observability 仪表盘——他在用一个叫 *Lyra-Curator* 的 agent
  整理自己一生的代码遗产,选出"配得上被记住的部分"。
  老人闭眼前问林行:"代码会被记住吗?"林行说:"会的,被它配得上的人记住。"

## 世界观要点(详见 worldbuilding.md · 待补)

- **"水位线"经济学**:推理成本 = 1/1000 一杯水
- **新职业**:skill writer / taste curator / context engineer / model whisperer
- **新基础设施**:个人 token 钱包、skill marketplace、"reasoning gas"期货
- **"代码遗产法案"**:政府开始讨论开源代码的"文化遗产"归属

## 风格

冷克制,长句压抑,短句砸地。
不写赛博朋克。不写末日。写的是**安静的崩塌**和**安静的重建**——
更像是村上春树的《海边的卡夫卡》遇见姜文的电影台词,而不是《黑客帝国》。
