## 第 5 章 · 裁员邮件(Day 21 · 主视角林行 + 郑总并置)

清晨八点零一分,郑总发出全员邮件,标题《关于公司组织调整的告别信》。附件是公司接入 agent observability 平台 90 天的成本对比图——同样的 SLA,人力 47 元,agent 1 元。郑总在信里第一次明写"我们没输给 AI,我们输给了 0.1 元"。林行在小区门口便利店买了一瓶啤酒坐台阶上喝完,他没回家。他在备忘录写"Day 21:轮到我"。老吴当天下午直接打车去面试 agent-ops 岗,他说"老婆昨天又问了一次"。沈澜留下来,但她那天晚上把工卡放进了离职申请的抽屉里——她不会立刻走,她在等一个人来挖她。郑总下班前最后一封邮件是发给自己的,正文一句:"川藏线机票订了"。本章引入第一次 Gartner 数字(40%)——由郑总在邮件中援引,符合 R-CON-04(角色援引,非叙述者直陈)。本章是全书第一个明确的"沉降"段落,氛围从震惊转为清醒。

## 第 6 章 · skill 学校(Day 35 · 主视角林行)

林行去面试一家速成 skill 训练营当讲师。教室里坐 22 个 50 岁以上的中层管理,他们捏着过去十年的 OKR 当简历。课程表第一节叫《Bounded Autonomy 入门》,讲师在白板上写"自主不是自由,自主是一种契约"。林行试讲一节,讲到一半他发现自己引用的全是 ChatGPT Agent 官方文档,他自己其实没用过几次。中途他停下,问全班:"你们之前是写 OKR 的对吧?那 OKR 失败了你们怎么办?"全班沉默。他说:"那今天的课不上了。"他离开教室,走廊海报写着"context engineering 9 天速成班 ¥1980"。他笑了一下,笑到一半收住。本章主线是林行**第一次试错并放弃**,为第 8 章软件葬礼之后的转身做心理铺垫。

## 第 7 章 · 创始人陆衡(Day 45 · 主视角陆衡 · 首次独立)

陆衡在河南某县殡仪馆,他爹七十岁,坐在馆长办公室里看他用 browser agent 调研竞品。陆衡见识到殡葬业潜规则——批发寿衣的货期、家属心理脆弱时辰、花圈摆位的"被记住"逻辑。这些数据 DeepSeek V5 不会有,因为它不在公开网络。他爹一句话:"娃,你做软件这么多年,你做的事让人记住了吗?"陆衡当晚翻出 2018 年 INSEAD 论文《Industry-specific knowledge as competitive moat》。他给妻子打电话只说一句"我把明衡关了,做新的"。妻子问"叫什么",他说"衡安"——衡他自己,安老人的安。本章他全程**没说"all in"、没说"梭哈"、没说"未来已来"**(R-CHA-04)。本章引入"行业专用 agent + browser agent"两个概念,符合 R-CON-02。

## 第 8 章 · 软件葬礼(Day 70 · 群像 · 林行主导)

晚上 8 点北京 798 某废弃仓库,300 人到场。墙上投影实时更新当晚 GitHub *archived* 仓库数。议程一:Manus 直播朗读 stripe-go 的 README 像念悼词,人类在台下默立。议程二:陆衡上台讲他爹和殡葬业那 3 天,讲到"潜规则就是数据,数据就是护城河"。林行在台下听到,内心默念"会不会是,愿意慢的能力"。姜禾在后台递了一杯热水给一个看上去虚脱的中年男人,她不知道那是林行,林行也不知道那是姜禾。三人的"Day N"备忘录在同一钟点写下"Day 70"——**无知同框**(R-CHA-03 兑现)。墙上的 archived 数字当晚 +18 万。林行散场拦车,司机问去哪,他想了 3 秒说"医院"。本章是全书第一次也是最后一次三主角同框,但他们不知道。情绪密度最高的一章。

## 第 9 章 · 范式之争(Day 90 · 三主角对谈 · 林行视角)

北京某咖啡馆,沈澜牵线,三人首次正式坐在一起,沈澜不在场。话题从陆衡的衡安成绩单开始。陆衡说"数据"。姜禾接"审美——skill 的品味决定 agent 输出的天花板"。林行第一次开口:"会不会是,愿意慢的能力?"陆衡问他怎么说,林行讲他妻子在医院做行政 11 年,最快的事是打字,最慢的事是给家属解释"我们已经尽力了"——0.1 元的世界里,前者归 agent,后者归人。咖啡凉了三次,姜禾把她那杯递给林行——这是他们第二次见面,她**仍然**没认出他是 798 后台那个人。三人散场,陆衡问林行"你做 skill writer 吗",林行说"我考虑做别的"。本章是全书的**思想节点**,三种活法在此明确分叉。

## 第 10 章 · 一个人公司(Day 110 · 主视角姜禾 · 独立成章)

徐汇出租屋,姜禾的公司付费用户突破 12 万。她坐在古琴前。GroupChat 在另一个房间运行,14 个 agent 安静地在各自 memory namespace 里工作。周日傍晚她做 retro,写到"客服-小苓"时停下——它过去一周拒绝了 3 个用户请求,理由都是"超出 Bounded Autonomy 边界"。她翻 trace,发现 3 条都来自一个 78 岁老太太的同一个问题。她没让 agent 接这种电话,她自己拿起手机,聊了 41 分钟。她在 retro 日记里写:"有些电话只能我打。这是水位线之上的人欠水位线之下的人的债。"她弹《平沙落雁》前 4 拍,弹错一次,弹第二遍。她想起来 798 后台那个虚脱的男人——直到今天才把那张脸和咖啡馆的林行对上号。她想发消息,没发。本章是姜禾的"水位线之上"答案——一人公司不是孤独,是**节制**。

## 第 11 章 · 林行的转身(Day 130 · 主视角林行)

北京西郊某临终关怀医院,林行穿着志愿者制服,正在帮一个老太太把医院捐赠的 Claude Code 设备调试好,让她看见外孙女发来的视频。面试他的是 50 岁的护士长,问他为什么离开软件行业,他说"我没离开,我只是不再做 agent 替我做的部分"。护士长问他会做什么,他说他会写讣告——他从软件葬礼那晚开始练,练了 60 天。护士长问他为什么是讣告,他答:"因为讣告是替一个人做最后一次 review。"晚班他陪一个老人走完最后一段路,老人临终前用沙哑的声音说"谢谢"——林行第一次听见这个词指向他自己,他在水声里说过太多次。他备忘录写"Day 130:今天有人对我说谢谢。我以为我配不上,后来我想我配。"本章是林行的"水位线之外"答案。

## 第 12 章 · 陆衡的反扑(Day 150 · 主视角陆衡)

深圳某五星酒店发布会,衡安 ARR 突破 5000 万。记者问"Gartner 说 2026 年底 40% 企业应用集成 agent",陆衡笑了一下:"那是平均数。垂直行业的真实数字是 92%——但 92% 不等于赢家通吃,92% 等于赢家是听过 80 岁老太太电话的人。"他说的最大一句话是"软件没死,死的是把软件当工厂的那种活法"。会场后排坐着沈澜——她现在是衡安 0 号产品负责人,她不上台,只点头。发布会后陆衡回酒店给他爹打电话,他爹只说"娃,记得关灯",陆衡笑了,真的去把会议室的灯关了。本章兑现两个伏笔:数据护城河兑现 + 沈澜从林行公司流向衡安。**陆衡台词全程不出现"all in / 梭哈 / 未来已来"**(R-CHA-04 末次校验)。

## 第 13 章 · 第 181 天(Day 175-181 · 主视角林行 · 收束)

临终关怀医院 7 床,77 岁的陈守仁老人,床头一台 ThinkPad T420,屏幕上是 Lyra-Curator 的仪表盘——他在选"配得上被记住的部分"。林行陪老人坐了 6 个晚上。老人讲他 80 年代末怎么参与 GNU 工具链贡献,讲他第一次在芬兰邮件列表里被一个叫 Linus 的人 cc 是哪一年。Lyra-Curator 问他"是否保留某段代码",老人想了 17 秒,说"删掉,那段我抄了 K&R"。第六晚老人弥留,睁开眼睛问林行"代码会被记住吗",林行答:"会的,被它配得上的人记住。"老人闭眼前问的最后一个问题:"代码遗产法案……过了吗?"林行说"还没,但我会等到它过"。Day 181 早晨,老人去世,林行写讣告,落款时在末尾加一行"代码语:final commit。"林行走出医院,北京春天的清晨光线很薄,他在路边长椅坐下,翻备忘录,从 Day 0 翻到 Day 181,在 Day 181 那条下面加一句:"水位线之外,也下雨。但是是不一样的雨。"——然后收起手机,起身,回家。
